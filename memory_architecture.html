Step 2 — 3 Storage Layers (expand each to explore)

🔷
FAISS Vector Store
Semantic similarity · Hot / Warm / Cold tiers
"What felt similar to this?"
▼

The message is converted into a 1024-dimensional vector using BGE-large-en-v1.5. This vector captures the meaning of the message — not the words, but the concept. Similar future queries will pull this memory even if they use completely different words.

HOT · < 30 DAYS
HNSW Index
Speed: ~5ms
Recall: ~98%
Compress: None
Precision: float32
Recent memories accessed constantly. No compression — full accuracy, maximum speed.
WARM · 30D – 1YR
IVF-SQ8 Index
Speed: ~15ms
Recall: ~95%
Compress: 4×
Precision: int8
Used occasionally. float32 → int8 saves 75% storage with minimal accuracy loss.
COLD · > 1 YEAR
IVF-PQ Index
Speed: ~25ms
Recall: ~90%
Compress: 8–16×
Precision: sub-vectors
Rarely accessed. Aggressive Product Quantization — 16× smaller. Old memories never deleted, just compressed.

→ Today's "quit job" memory enters the HOT tier. After 30 days → WARM. After 1 year → COLD. Always retrievable.

🟡
DuckDB Metadata Store
Structured SQL · Time · Topic · Emotion
"When did this happen?"
▼

FAISS finds similar memories but cannot answer "show me everything from last March" or "find all moments I felt anxious." DuckDB stores every memory as a structured SQL row — filterable by any field.

INSERT INTO memories (
  id,      -- unique ID
  content,  -- "quit my job today..."
  type,     -- episodic / semantic / reflective / belief
  timestamp,-- 2026-03-10 14:23:07
  emotion,  -- "excited-anxious"
  importance,-- 0.94 (high life event)
  topics,   -- ["career", "startup", "decisions"]
  entities, -- ["Suraj", "job", "startup idea"]
  causes,   -- ["startup felt real"]
  effects   -- ["left employment"]
);

→ Query: SELECT * FROM memories WHERE 'career' = ANY(topics) ORDER BY timestamp
→ Returns the full career timeline — every related memory, in order.

🟢
NetworkX Knowledge Graph
Entity relationships · Causal edges · Temporal chains
"Why did this happen?"
▼

This is Cortex Lab's most unique layer. It stores relationships between entities as a directed graph. Each edge has a type and timestamp — building a permanent map of cause and effect across your entire life.

From today's message, these edges are created:

Suraj
──[quit]──▶
job
Suraj
──[believes_in]──▶
startup idea
──[caused]──▶
quit job

+ Previous edges already in graph:
Suraj ──[was_frustrated_by(3mo ago)]──▶ job
Suraj ──[explored(6mo ago)]──▶ startup idea

→ Now the system can answer "Why did I quit?" with real causal chains spanning months.

Query: "What led to my career change?"

Watch all 3 layers contribute to a single answer

1

FAISS — Semantic Recall

Finds all memories with vectors close to "career change" — surfaces quit job, job frustration, startup exploration even without exact keyword matches. Returns top-20 by cosine similarity.

2

DuckDB — Timeline Construction

SQL query: SELECT * FROM memories WHERE 'career' = ANY(topics) ORDER BY timestamp ASC — returns the precise chronological sequence of every career-related memory.

3

NetworkX — Causal Path Traversal

Graph traversal from job → follows causal edges backwards → finds frustration (3mo) → startup exploration (6mo) → belief shift → quit decision. Multi-hop reasoning across months.

4

RRF Fusion + Reranking

All 3 layers' results merged via Reciprocal Rank Fusion. BGE cross-encoder reranker selects the most relevant pieces. Combined latency: ~100ms.

5

DeepSeek-R1-7B → Final Answer

CausalAgent calls LocalLLM.trace_causal_chain() with fused context. Generates narrative explanation.

Cortex Lab Response

Over 8 months, two forces compounded: growing frustration with your job (first noted 6 months ago) and an expanding belief in your startup idea (explored 6 months ago, confidence rising steadily). On March 10, 2026, the startup conviction crossed a threshold — making the cost of staying feel greater than the risk of leaving. The causal chain: frustration + conviction → decision → action.
